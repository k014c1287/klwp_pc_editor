# KLWP Desktop Editor — 使い方

Windows 上で `.klwp` を開いて編集し、Android の KLWP に読み込ませるためのスタンドアロンエディタです。

## プロジェクト構成

```
klwp_editor.py          23行の起動・後方互換ファサード
klwp/                   アプリケーション本体（責務別パッケージ）
  archive.py            .klwp ZIP/JSON境界
  background.py         背景画像の数式・Bitmap Global紐付け
  adb.py                Android端末検出・adb転送
  formula.py / svg.py   Kode評価 / SVGパス解析
  preview/              ページ・Switch・アニメーション状態
  render/               配置・図形・文字・合成描画
  ui/                   ウィンドウ・各編集ダイアログ・操作
test_klwp_editor.py     機能・描画回帰テスト
test_architecture.py    リファクタリング規約の静的テスト
tools/                  開発用規約チェッカー
README.md               利用方法（このファイル）
doc/                    引き継ぎ資料・開発文書
  KLWPエディタ_設計仕様.md  Mermaidクラス図・シーケンス図
sample/                 KLWP成果物と比較用スクリーンショット
```

READMEはプロジェクトルート、開発文書は `doc/`、KLWPの正解データと関連スクリーンショットは `sample/` に配置します。

## 1. セットアップ（初回のみ）

1. Python をインストール（3.9 以上、https://www.python.org/ から。「Add python.exe to PATH」にチェック）
2. コマンドプロンプトで画像プレビュー用ライブラリを導入:
   ```
   pip install pillow
   ```
3. 起動:
   ```
   python klwp_editor.py
   ```

## 2. exe 化（Python なしで起動したい場合）

```
pip install pyinstaller
pyinstaller --onefile --windowed klwp_editor.py
```
`dist\klwp_editor.exe` が生成されます。以後はダブルクリックで起動できます。

## 3. 画面と機能

| 場所 | 機能 |
|---|---|
| ツールバー | 新規 / 開く / 保存、Android転送、元に戻す / やり直す、要素追加、複製・削除・前面／背面移動、背景・画像・グローバル・プレビュー値管理 |
| 左ペイン | 要素ツリー。種類と前面順を表示し、同じレイヤー内の要素をドラッグして表示優先度を変更できます。「選択解除（ルート）」またはツリーの空白クリックで追加先をルートへ戻せます |
| 中央 | プレビュー。要素をドラッグで移動し、選択した図形・画像の縁やハンドルをドラッグしてサイズ変更できます |
| 右ペイン | 選択要素のプロパティ（名前・アンカー・オフセット／四辺余白・サイズ・色・フォント等）。アンカーは9方向の日本語プルダウン、色はカラーピッカーで編集可能 |

- 色はカラーピッカー、色見本、不透明度（0–100%）で直感的に指定できます。KLWP形式 `#AARRGGBB` の直接入力も使用できます
- Undo は `Ctrl+Z`、Redo は `Ctrl+Y` または `Ctrl+Shift+Z` でも実行できます（最大50操作）
- 「詳細（JSON を直接編集）」で、プロパティ欄にない設定（アニメーション、タッチアクション、数式 `internal_formulas` 等）もすべて編集できます
- 「画像管理」で内蔵画像の一覧・プレビュー・差し替えができます（● は使用中）
- 「背景設定」で固定画像に加え、BITMAP型Globalの選択、背景用画像Globalの追加、`background_bitmap` のKode数式を編集できます
- 「グローバル管理」はルートGlobalを編集します。ローカルGlobalを持つKomponentでは、右ペインに専用ボタンも表示されます
- 「＋図形」はプルダウンから図形種別を選び、追加前に色と不透明度も指定できます
- 図形は上下左右を自由にリサイズできます。画像は元画像の縦横比を維持してリサイズされ、重ねレイヤー内の子要素にも対応します
- レイヤー選択中は新規要素がそのレイヤー内へ追加されます。ルートへ追加する場合は「選択解除（ルート）」、ツリーの空白、または `Esc` で選択を解除します

### Switch・アニメーション・タップ設定

1. ツールバーの「グローバル管理」で型に `SWITCH` を選び、Switchグローバルを作成します
2. 対象要素を選び、右ペインの「アニメーション設定」で `SWITCH` と作成したSwitchを紐づけます
3. タップさせる要素の「タップイベント設定」で `SWITCH_GLOBAL` と同じSwitchを選びます
4. 「操作プレビュー」をONにしてタップ動作を確認します

アニメーションは複数追加でき、SCROLL / SWITCH / LOOP_2Wと、移動・反転移動・フェード・回転・拡縮・色フィルターを設定できます。補間は直線・加速・減速・オーバーシュート・バウンスに対応し、既存の未知アクションも保持します。

### 操作・アニメーションプレビュー

- 中央下の「操作プレビュー」をONにすると、キャンバスのドラッグは編集ではなく横スワイプになります
- 横スワイプまたは「ページ」スライダーで、`SCROLL` の移動・フェードが連続的に追従します
- `SWITCH_GLOBAL` が設定された要素をタップすると、対応する `SWITCH` アニメーションとグローバル数式が切り替わります
- Nova等の `PAGE_NUMBER` 付きページ移動ショートカットは、PC内のページ移動として再現します
- 「ループ再生」で `LOOP_2W` の各対応アニメーションを確認できます
- `LAUNCH_APP`、ショートカット、URI、音楽操作、Kustom動作をタップ設定UIから作成できます。PCでは安全のため外部実行しません
- 「プレビュー値」で日時・バッテリー・天気・音楽・位置・通信等のKode入力値を変更できます（成果物には保存しません）
- 日時を変更すると、背景数式が参照するBITMAP型Globalも再評価され、時間帯別の背景を確認できます

## 4. プレビューの注意（重要）

PC 上のプレビューは**近似表示**です。Bitmap、PATH、Komponent倍率、回転、外側グロー、
すりガラス、グラデーション、主要blend mode、Stack整列、主要な `internal_formulas`、
SCROLL/SWITCH/LOOP_2Wアニメーションを描画します。
以下は Android 実機での最終確認が必要です。

- 時刻・天気・バッテリー・音楽等は編集可能な模擬値であり、Androidの実データとは連携しません
- Android/Pillow間で画素計算が異なる高度なグラデーション、blend、色フィルター
- 未対応のKode関数・新しいShader・複雑なアニメーション
- 数式で切り替わるFontIconは、アーカイブに保存されたSVG形状を使用
- Zen Kaku Gothic 等の実フォント描画

「PC で構造と配置を作り、実機で最終確認」という流れを想定しています。

## 5. Android への転送と読み込み

ツールバーの「Androidへ転送」は、保存後に接続端末を検出して
`/sdcard/Kustom/wallpapers/` へワンクリック転送します。Android SDK Platform-ToolsとUSBデバッグが必要です。

手動転送方法はどれでも構いません:

1. **USB ケーブル**: エクスプローラーで端末の `内部ストレージ/Kustom/wallpapers/` にコピー
2. **Google Drive / Dropbox**: アップロードして端末でダウンロード
3. **adb**（開発者向け）:
   ```
   adb push mypreset.klwp /sdcard/Kustom/wallpapers/
   ```

読み込み手順（KLWP アプリ側）:

1. KLWP を開く → 左上メニュー → 「読み込み...」
2. `Kustom/wallpapers` に置いた場合は一覧に表示されます。別の場所なら「SD カード」からファイルを選択
3. 保存（フロッピーアイコン）→ 壁紙として適用

※ 編集済みプリセットの保存・書き出しには KLWP Pro（有料版）が必要です。読み込みと適用は無料版でも可能です。

## 6. 既知の制限

- 要素の新規追加はテキスト・図形・アイコン・画像・レイヤーの5種（その他は JSON 編集で追加可能）
- Kode評価器は `sample/` で使われる全関数と主要な数学・文字列・色関数に対応しますが、公式Kode全関数ではありません
- Komponent倍率、グラデーション、blend、複雑なアニメーションはPillowによる近似表示です
- 外部アクションはIntent/URIを入力できますが、PCからAndroidのインストール済みアプリ一覧は取得しません
- KLWP 側で作った複雑な設定は保持されます（編集しない項目はそのまま書き戻されるため、既存プリセットを壊しません）
- 公式v1/v3/v4/v5と実成果物v10/v11/v15を回帰対象にしていますが、未知形式は実機でも確認してください

## 7. テスト

```
python -m unittest -v
python tools/check_object_calisthenics.py
```

sample内のKLWPについて、アーカイブ往復保存、画像ID・ZIP互換性、v1/v3/v4/v5/v10/v11/v15、Kode、Bitmap比率、数式・BITMAP Globalによる時間帯別背景、全図形タイプ、グラデーション・blend、Komponent倍率、アンカー基準のオフセットと四辺余白、直接リサイズ、要素ツリーの選択解除とドラッグ並べ替え、Undo/Redo、タップ、各種アニメーション、ADBコマンドを検証します。現在は機能50件とアーキテクチャ1件の計51テストです。

規約チェッカーは `klwp/` と `tools/` を対象に、`else` 禁止、メソッド内ネスト1段、1メソッド30行以内、1クラス250行以内、1クラスのインスタンス変数2個以内、property/getter/setterデコレータ禁止、二段以上のメッセージ連鎖禁止を検証します。Tkinter・Pillow・JSONへ渡す生の値は境界に限定し、アプリ内部の状態は値オブジェクトとファーストクラスコレクションで扱います。
